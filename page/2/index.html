Maven知识整理

發表於 2015-08-04   |   分類於 Maven

maven快速入门

  1. maven介绍以及环境搭建
    maven是基于项目对象模型(pom),可以通过一小段描述信息来管理项目的构建、
    报告、和文档的软件项目管理工具。覆盖编译，打包，清理等周期。
    maven下载地址
    下载解压后有四个文件夹。bin中是一些mvn的运行脚本。在命令行中可以用mvn进行调用这些脚本。
    boot下面是一个maven的类加载器的框架。config下面是配置文件，例如主要的settings配置文件。
    lib目录下面是maven常用的一些jar。
  2. 配置环境变量
    在系统变量中新建一个M2_HOME路径为maven的安装目录。
    然后在path中加入%M2_HOME/bin。然后在命令行中输入mvn -v来查看是否安装成功。
  3. maven pom文件中几个属性值
    groupId为项目的包名;
    artifactId是模块名（项目名）;
    version SNAPSHOT就是快照（一般用0.0.1SNAPSHOT）;

maven核心知识

  1. 常用的构建命令
    mvn- v 查看版本
    compile 编译
    test 测试
    package 打包
    clean 删除target
    install 安装jar到本地仓库中
  2. 自动创建目录骨架 archetype插件
    mvn -archetype generate 下载插件
    按照提示去创建，也可以直接在后面跟上 -DgroupId -DartifactId -Dversion -Dpackage
  3. maven中的坐标和仓库
    坐标：groupId为项目的包名
    artifactId是模块名（项目名）
    version 版本
    仓库：本地
    远程(lib目录下面的maven-model-bulider.jar中pom文件中)
    镜像仓库：settings.xml中的mirrors中的mirror
    更改仓库位置：settings.xml中localRepository属性进行配置更改
    例如改为E:/myMavenStore,然后将settings文件放到当前文件夹中，方便以后修改管理
  4. 在eclipse4.0以后，将maven插件是内置的不需要下载。如果用以前的版本，可以下载插件，不再赘述。
    在eclipse中选择新建maven项目有两种。
    maven-archetype-quickstart
    默认的Archetype,基本内容包括：
    一个包含junit依赖声明的pom.xml
    src/main/java主代码目录及一个名为App的类
    src/test/java测试代码目录及一个名为AppTest的测试用例
    maven-archetype-webapp
    一个最简单的Maven war项目模板，当需要快速创建一个Web应用的时候可以使用它。生成的项目内容包括：
    一个packaging为war且带有junit依赖声明的pom.xml
    src/main/webapp/目录
    src/main/webapp/index.jsp文件
    src/main/webapp/WEB-INF/web.xml文件

  5. maven的生命周期和插件
    完整的项目构建过程包括：
    清理，编译，测试，打包，集成测试，验证，部署
    maven的生命周期：

    clean   清理项目
default 构建项目
compile test package install
site    生成项目站点
pre-site 在生成站点前完成的工作
site 生成项目的站点文档
post-site 在生成项目站点后要完成的工作
site-deploy 发布生成的站点到服务器上

    maven的插件有很多。比如将项目源码打包的source，还有代码检测的findbugs等等。
    具体配置例如：

    `<plugin>
    <groupId>org.codehaus.mojo</groupId>
        <artifactId>findbugs-maven-plugin</artifactId>
            <version>${findbugs-maven-plugin.version}</version>
            <executions>
                <execution>
                <!--阶段-->
                    <phase>verify</phase>
                <!--目标-->
                    <goals>
                         <goal>check</goal>
                    </goals>
                </execution>
            </executions>
</plugin>`

    pom中的标签
    详解http://blog.csdn.net/sunzhenhua0608/article/details/32938533

  7. 依赖范围
    控制依赖与具体classpath的关系
    scope ：
    compile 默认的范围，编译，测试运行都有效
    provided 编译和测试时有效
    runtime 测试和运行时有效
    test 测试的时候有效
    system 编译和测试时有效，要与本地项目相关联
    import 只用在dependencyManagement中，表示从其他的pom中导入dependency的配置
  8. 依赖传递
    A>B>C
    B依赖A，C依赖B
    在B中配置，
    `<dependencies>
        <dependency>
            <groupId>A</groupId>
            <artifactId>A</artifactId>
        </dependency>
</dependencies> `
    在C中配置
    `<dependencies>
        <dependency>
            <groupId>B</groupId>
            <artifactId>B</artifactId>
        </dependency>
</dependencies>`
    但是这样存在一个问题，由于依赖存在传递性，其实C中也存在A，但是C不需要依赖A，只需要依赖B即可
    所以我们可以加上如下的配置：
    `<dependencies>
        <dependency>
            <groupId>B</groupId>
            <artifactId>B</artifactId>
            <exclusions>
            <exclusion>
            <groupId>A</groupId>
            <artifactId>A</artifactId>
            </exclusion>
            </exclusions>
        </dependency>
</dependencies>`
  9. 依赖冲突
    依赖冲突就是说A，B依赖了不同的版本一个构建，但是C又同时依赖A,B.这样就导致了版本冲突。
    解决的方案：
    9.1 短路优先  依据读取的路径，依赖离得最近的优先解析。
     举例：A>B>C>X.jar
           A>B>X.1.jar
     优先解析的是   A>B>X.1.jar
9.2 先声明先优先  如果路径也相同，那就是看配置的先后顺序
  10. 聚合和继承
    10.1 聚合
     如果将多个项目install安装到本地仓库中，对此依次执行，但是可以有聚合的方式，一次执行它。
     在聚合项目的pom中将最上面的package改为pom。不用dependencys 而是使用modules 然后将install的项目配置的module标签中。
     (eclipse可以自动提示)
10.2 继承 
     使用parent标签。

Java中的网络支持

根据网络的不同层次，Java提供的网络功能有四大类

1.InetAddress :用于标识网络上的硬件资源。
2.URL :统一资源定位符 通过URL可以直接读取或者写入网络上面的数据。
3.Sockets :使用TCP协议实现网络通信的Scoket相关的类。
4.Datagram :使用UDP协议 将数据保存在数据报文中 通过网络进行通信。

InetAddress类

InetAddress没有构造方法，所以我们不能直接new出一个对象。但是api上面可以看出，InetAddress
有很多的静态方法，返回的实例就是InetAddress的。通过InetAddress的实例，我们可以得到计算机名，
IP地址等。

URL

URL由两部分构成，协议名称和资源名称，中间用冒号隔开。java中获得URL的实例可以通过构造方法去new
出来。

杂七杂八（一）

　　今年公司提出每天多少要写点代码分析，以及每周五进行代码分享。这个建议不错，知识那么多，好记性不如烂笔头，每天记录些还是很有好处的。自己也写了很多篇代码分析了，都是自己在平时工作学习中总结或者解决的一些知识。我的初衷还是想写点真正有质量的，网上不是随地可见的知识分享给大家的。但是由于自己身上的惰性，很多东西领悟了但是没有很好的记录下来，也没有很好的巩固，所以有些比较难的地方，说的不是很透彻。我是相当愿意将知识很通俗的解释给别人听的，比如有时候测试的同事就会问一些web应用配置上面的问题，因为有些是设置spring的一些知识，所以要给人讲明白讲清楚，这样就会用一些通俗的语言去解释这些东西。所以知道再讲出来，再到讲出来大家都能理解。这是一个锻炼自己的过程。还有一些知识是自己没有学习过的，在平时的工作中也没有遇到过，写出来的话，能够抛砖引玉，让别的同事来指导一下，请教会这种知识的人，也未尝不是一件乐趣啊。
　　所以，要多写写，写出来给大家看，文章中有不对的地方也能按照别人的建议修改，及时纠正错误。这是一种相互学习的过程。
