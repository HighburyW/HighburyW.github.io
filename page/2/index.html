数据库迁移框架——Flyway

發表於 2015-09-11   |   分類於 java

实施部署

發表於 2015-09-01   |   分類於 java

在部署系统的时候，oracle是在集群上面。所以我们原先用使用sid来配置jdbcurl已经不行了，会导致服务启动不起来，因为对于集群来讲，每一个节点的sid都是不一样的。所以我们将原先的格式修改为集群下的配置，将sid来替换为service_name ，service_name 包含所有的节点。
原来的配置：
<Set name="jdbcUrl">jdbc:oracle:thin:@172.16.2.69:1521:ahltollglc</Set>
修改为：
<Set name="jdbcUrl">jdbc:oracle:thin:@//172.16.2.69:1521/ahltollglc</Set>
注：@ 后面加上//这是与使用sid的主要区别。
其实oracle连接java jdbc/ojdbc 驱动的方式一共有三种。第一种是Oracle Thin JDBC Driver，第二种是Oracle OCI JDBC Driver，第三种是Oracle 8i OCI JDBC Driver 。上面说的那两种配置的方式其实都是使用的Oracel Thin JDBC Driver这种驱动的方式，这种还有一种形式就是用数据库的TNSName来配置，如：
jdbc:oracle:thin:@TNSName
注：TNSName是本地的名称，这种方式我没有进行配置，因为这是本地命名服务配置的。估计远程的话这种方式应该是连接不上的，但是没有经过测试。
像Oracle OCI JDBC Driver这种方式配置url的方式为：
jdbc:oracle:oci:@port/service_name
//或者
jdbc:oracle:oci:@port:sid
oci这种形式是一种胖客户端的连接方式，而thin是一种瘦客户端的形式。用oci的话需要本地安装oracle的客户端，thin的话就不需要，所以大多数都是用的thin这种连接方式。最后还有一种是8i OCI这种驱动连接的方式，配置的方式为：
jdbc:oracle:oci8:@TNSName
这种需要引进classes12.zip这个文件。
以上就是在工作中总结的oracle的几种连接的方式。推荐的还是用service_name这种形式，官方现在也是推荐用这种形式，不再用sid这种去配置url。

关于spring配置文件中resourceRef的配置

發表於 2015-08-27   |   分類於 java

关于spring配置文件中resourceRef的配置。收费站和管理处的程序，有些是部署在tomcat，有些是部署在weblogic上面。但是在不同的容器中resourceRef的配置是不同的，我们在代码中默认是false，但在tomcat上面要设置true，这样才能保证服务能够正常的启动。
在tomcat上面：
<bean id="dataSource" class="org.springframework.jndi.JndiObjectFactoryBean">
<property name="resourceRef">
<value>true</value>
</property>
<property name="jndiName">
<value>jdbc/toll</value>
</property>
</bean>
我查了一下JndiObjectFactoryBean中的关于resourceRef的描述，在JndiLocatorSupport的抽象类中有这样的解释:

/**
* Set whether the lookup occurs in a J2EE container, i.e. if the prefix
* "java:comp/env/" needs to be added if the JNDI name doesn't already
* contain it. Default is "false".
* <p>Note: Will only get applied if no other scheme (e.g. "java:") is given.
*/

所以在tomcat中设置jndi的时候需要加上java:comp/env/，因此要设置为true，因为设置成为true之后，程序会自动添加java:comp/env/给jndi。设置为true对weblogic，jetty这些容器也不会产生影响，这样的话，我们在下收费站实施部署的时候，就不用考虑容器是什么而去修改这些配置文件了。

jndi多数据源配置

發表於 2015-08-21   |   分類於 java

今天早上跟领导讨论下关于报表服务启动一个，同时由稽查和收费两个系统访问的问题。这样就省的再启动一个多余的report服务，节省了服务器资源。所以我用中午的时间做了一下测试，验证了一下。其实这个主要的问题还是在于是否支持多数据源，因为稽查和收费不在同一个数据库中。查看下fineReport的api，这个工具是能够同时支持多个数据源的。然后再看spring,spring对多数据源也能够很好的支持。所以问题就迎刃而解了。下面是测试的步骤：
1.配置applicationContext.xml。

<!-- 收费-->
<bean id="dataSource" class="org.springframework.jndi.JndiObjectFactoryBean">
<property name="resourceRef">
<value>true</value>
</property>
<property name="jndiName">
<value>jdbc/toll</value>
</property>
</bean>
<!-- 稽查-->
<bean id="auditDataSource" class="org.springframework.jndi.JndiObjectFactoryBean">
<property name="resourceRef">
<value>true</value>
</property>
<property name="jndiName">
<value>jdbc/audit</value>
</property>
</bean>

这里面report项目中并不需要jdbctemper之类的，所以不需要来引用datasource。如果在别的项目中的话，需要重写下AbstractRoutingDataSource的determineCurrentLookupKey方法来获得不同的数据源。这个类的父类AbstractDataSource是spring获得数据源的类。今天只是测试report的服务，所以没再继续看这块。

Java删除ArrayList中重复的元素

發表於 2015-08-04   |   分類於 java

Java删除ArrayList中的重复元素的2种方法

ArrayList是我们经常使用的集合类型之一。它允许我们添加多个null元素，重复的元素，还能够保持插入的顺序。
在具体学习和工作中我们经常会遇到需要删除ArrayList中重复的数据。下面我将代码举例探讨下两种删除的方法。

方法1：使用HashSet删除ArrayList中重复的元素

在该方法中，我们使用HashSet来删除重复的元素。如你所知，HashSet不允许有重复的元素。我们使用HashSet的
这个属性来删除已建成的ArrayList中的重复元素。但是，这种方法有一个缺点。那就是，它会删除ArrayList中元
素的插入顺序。这意味着，删除重复的元素后，元素的插入顺序就不对了。先来看下面这个例子。

`import java.util.ArrayList;
import java.util.HashSet;

public class MainClass
{
    public static void main(String[] args)
    {
        //Constructing An ArrayList

        ArrayList<String> listWithDuplicateElements = new ArrayList<String>();

        listWithDuplicateElements.add("JAVA");

        listWithDuplicateElements.add("J2EE");

        listWithDuplicateElements.add("JSP");

        listWithDuplicateElements.add("SERVLETS");

        listWithDuplicateElements.add("JAVA");

        listWithDuplicateElements.add("STRUTS");

        listWithDuplicateElements.add("JSP");

        //Printing listWithDuplicateElements

        System.out.print("ArrayList With Duplicate Elements :");

        System.out.println(listWithDuplicateElements);

        //Constructing HashSet using listWithDuplicateElements

        HashSet<String> set = new HashSet<String>(listWithDuplicateElements);

        //Constructing listWithoutDuplicateElements using set

        ArrayList<String> listWithoutDuplicateElements = new ArrayList<String>(set);

        //Printing listWithoutDuplicateElements

        System.out.print("ArrayList After Removing Duplicate Elements :");

        System.out.println(listWithoutDuplicateElements);
    }
}
`

输出结果：
ArrayList With Duplicate Elements :[JAVA, J2EE, JSP, SERVLETS, JAVA, STRUTS, JSP] ArrayList After Removing Duplicate Elements :[JAVA, SERVLETS, JSP, J2EE, STRUTS]

注意输出结果。你会发现，在删除重复元素之后，元素重新洗牌。不再按照插入顺序排列。如果你想在删除重复的元素
之后依然保持元素的插入顺序，那么不建议使用此方法。还有另一种方法，可以保证在删除重复的元素之后也不改变元
素的插入顺序。那就是使用LinkedHashSet。

方法2：使用LinkedHashSet删除ArrayList中重复的元素

在该方法中，我们使用LinkedHashSet删除ArrayList中重复的元素。正如你知道的，LinkedHashSet不允许重复元素，同
时保持元素的插入顺序。LinkedHashSet的这两个属性可以确保在删除ArrayList中的重复元素之后，依然保持元素的插
入顺序。参见下面的例子。

`import java.util.ArrayList;
import java.util.LinkedHashSet;

public class MainClass
{
    public static void main(String[] args)
    {
        //Constructing An ArrayList

        ArrayList<String> listWithDuplicateElements = new ArrayList<String>();

        listWithDuplicateElements.add("JAVA");

        listWithDuplicateElements.add("J2EE");

        listWithDuplicateElements.add("JSP");

        listWithDuplicateElements.add("SERVLETS");

        listWithDuplicateElements.add("JAVA");

        listWithDuplicateElements.add("STRUTS");

        listWithDuplicateElements.add("JSP");

        //Printing listWithDuplicateElements

        System.out.print("ArrayList With Duplicate Elements :");

        System.out.println(listWithDuplicateElements);

        //Constructing LinkedHashSet using listWithDuplicateElements

        LinkedHashSet<String> set = new LinkedHashSet<String>(listWithDuplicateElements);

        //Constructing listWithoutDuplicateElements using set

        ArrayList<String> listWithoutDuplicateElements = new ArrayList<String>(set);

        //Printing listWithoutDuplicateElements

        System.out.print("ArrayList After Removing Duplicate Elements :");

        System.out.println(listWithoutDuplicateElements);
    }
}`

输出：
ArrayList With Duplicate Elements :[JAVA, J2EE, JSP, SERVLETS, JAVA, STRUTS, JSP] ArrayList After Removing Duplicate Elements :[JAVA, J2EE, JSP, SERVLETS, STRUTS]

这样，我们就保持了ArrayList的原有插入顺序。但是同样需要注意的一点是LinkedHashSet在迭代访问Set中的全部元素时，
性能比HashSet好，但是插入时性能稍微逊色于HashSet。看具体操作的时候需要那种结果。
